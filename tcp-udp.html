TCP vs UDP

Both TCP and UDP sit at the Transport layer and both use port numbers to reach the right app — but they make opposite trade-offs. TCP spends time and overhead guaranteeing delivery. UDP skips all of that for speed.

ClientServer

Side by side

TCPUDP
ConnectionSet up first (3-way handshake)None — just send
ReliabilityLost data is detected & resentLost data just... stays lost
OrderingReassembled in the right orderNot guaranteed
Speed / overheadSlower, more overheadFast, minimal overhead
Typical useWeb pages, email, file transferVideo calls, live streaming, DNS
UDP isn't "worse" — it's a deliberate choice. A dropped word in a live video call is barely noticeable and resending it late would be pointless; but a dropped chunk of a downloaded file would corrupt it. Each protocol fits different traffic.
