DNS & HTTP

Type a URL and hit enter — before a single pixel of the page loads, your browser has to translate a human-readable name into an IP address (DNS), then have an entire conversation with that server (HTTP). Run it below.

The cast of characters

ActorRole
ResolverUsually run by your ISP or a public service (like 1.1.1.1) — does the lookup legwork for you and caches answers
Root serverKnows which servers handle each top-level domain (.com, .org, etc.)
TLD serverKnows which nameserver is authoritative for a specific domain
Authoritative nameserverHolds the actual, final answer: the IP address for that domain
Caching is why the second visit to a site is faster than the first — your resolver already has the IP address memorized and skips straight to the HTTP request.
