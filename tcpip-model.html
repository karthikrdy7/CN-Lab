TCP/IP Model

OSI is the teaching model. TCP/IP — sometimes called the Internet Protocol Suite — is the model the actual internet runs on, and it only has 4 layers. It groups several OSI layers together because, in practice, real protocols don't respect OSI's clean boundaries. Hover a row to see how the two line up.

TCP/IP (4 layers)

Why it has fewer layers

OSI separated Application, Presentation, and Session because, in theory, those are three distinct jobs. In practice, a single protocol like HTTP handles formatting and session behavior itself — there's no separate "presentation protocol" running underneath it. TCP/IP just admits that and merges all three into one Application layer.

When people say "which layer does a firewall work at?" or "that's a layer 3 problem," they're almost always using OSI's numbers — even on a network that's technically running the TCP/IP model. OSI survives as the shared vocabulary.
