Switching (Layer 2)

A switch connects several devices on the same local network and forwards traffic between them using MAC addresses — hardware IDs burned into every network card. A switch starts out knowing nothing, and builds up a MAC address table by watching where traffic comes from.

The rule the switch follows

SituationSwitch behavior
Frame arrivesAlways learn: record (source MAC → the port it arrived on)
Destination MAC knownUnicast — forward out only that one port
Destination MAC unknownFlood — forward out every port except the one it arrived on
This is why the very first frame to a new device always gets flooded — the switch has no way to know where it is yet. Every reply teaches the switch a little more, until eventually it knows exactly where every device sits.
