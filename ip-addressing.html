IP Addressing & Subnets

An IPv4 address is 32 bits, usually written as four decimal numbers (0–255) separated by dots. A subnet mask — or the /24-style CIDR notation — decides how many of those 32 bits identify the network and how many identify a specific host on it.

/

How to read it

Take 192.168.1.50/24. The /24 means the first 24 bits (the first 3 octets: 192.168.1) are the network portion — fixed for every device on this subnet. The remaining 8 bits (the last octet) identify one specific host, and can range from 0–255, giving 256 possible values. Two of those are reserved (the network address itself, and the broadcast address), leaving 254 usable host addresses.

A smaller CIDR number (like /16) means fewer network bits and more host bits — a bigger subnet with room for more devices. A larger CIDR number (like /30) means almost no room for hosts, often used for point-to-point links between two routers.
